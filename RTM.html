Airvue UI/UX Traceability Matrix

A comprehensive mapping of requirements to the current UI/UX implementation for the Airvue air quality monitoring platform.

Overview

This traceability matrix maps the requirements from the project phases document to the current UI/UX implementation as seen in the available screens. It serves as a guide for the UI/UX developer to understand what has been implemented and what still needs to be developed.

Current Screens Available for Analysis:

  • Main Dashboard - Air quality metrics, device AQIs, pollutant values, and calendar
  • Add Device - Form for adding new monitoring devices
  • Devices Listing - Table of all devices with status and actions
  • Add User - Form for adding new users with permissions
  • Users Listing - Table for user management (currently empty)
  • Profile - User profile and password management

Gaps and Recommendations

High Priority Gaps

  • Subscription Management: No UI for subscription plans or payment processing
  • Device Configuration: Missing UI for sensor parameter configuration
  • Data Export: No options for exporting data in different formats
  • Historical Data Visualization: Limited implementation of historical data views

UI/UX Recommendations

  • Add subscription management section under user settings
  • Implement device configuration options in device detail view
  • Add export buttons to dashboard and data views
  • Enhance historical data visualization with filtering and comparison tools
  • Implement map view for device tracking
  • Add compliance reporting section for certification metrics

Next Steps for UI/UX Development

  • Complete the Phase 1 features with missing UI elements
  • Design mockups for Phase 2 enhancements
  • Develop user flow diagrams for Mobile App requirements
  • Create wireframes for AI insights visualization
